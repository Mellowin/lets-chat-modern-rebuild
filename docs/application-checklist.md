# Application Checklist — lets-chat-modern-rebuild

Practical checklist to start applying for Junior Backend / Full-stack Junior roles.

---

## Before Applying

- [ ] GitHub repo `lets-chat-modern-rebuild` is pinned on my profile.
- [ ] Repo short description and topics are filled in.
- [ ] Profile README includes a featured-project block.
- [ ] LinkedIn / Djinni / DOU profile is updated with project link and stack.
- [ ] Resume includes the project block from `docs/resume-project-blocks.md`.
- [ ] Live demo loads: https://lets-chat-web.vercel.app
- [ ] API health returns `ok`: https://lets-chat-api-v2.onrender.com/api/v1/health
- [ ] README screenshots render correctly.
- [ ] Recruiter messages are copied and placeholders replaced.
- [ ] 30-second pitch is rehearsed.
- [ ] 2-minute demo story is rehearsed.
- [ ] Interview answers are reviewed.

---

## Interview Browser Tabs to Have Ready

Keep these open before a screen-share interview:

1. Live app: https://lets-chat-web.vercel.app
2. GitHub repo: https://github.com/Mellowin/lets-chat-modern-rebuild
3. README → screenshots section.
4. Latest green CI run: https://github.com/Mellowin/lets-chat-modern-rebuild/actions
5. Project story or demo script: `docs/project-story.md` or `docs/demo-script.md`
6. Job application kit: `docs/job-application-kit.md`

---

## Daily Routine

- [ ] Send 5–10 targeted applications per day.
- [ ] Track each application: company, role, link, date, status.
- [ ] Customize the first line of each recruiter message for the company/role.
- [ ] Follow up after 5–7 days if no response.
- [ ] Update resume or profile based on feedback.
- [ ] Wake up the backend before a demo: https://lets-chat-api-v2.onrender.com/api/v1/health
